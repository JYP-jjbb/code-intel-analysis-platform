2) Grammar (informal):

SEMANTIC TARGET (do not explain in output):
- RF is non-negative on reachable loop states.
- RF strictly decreases on every loop iteration.
- If scalar is hard, use a lexicographic vector (2D–6D preferred).

SEARCH STRATEGY (important, do not explain in output):
- Prefer structure-driven candidates over random constants.
- Diversity checklist (MUST progress; do NOT only tweak constants/order):
  1) Single-variable decrease: ReLU(x) / ReLU(x - L)
  2) Distance-to-upper-bound: ReLU(U - x)
  3) Difference convergence: ReLU(x - y) / ReLU(y - x)
  4) Absolute-value style: ReLU(x) + ReLU(-x)
  5) Multi-phase / branch split: 3–6D lex vector, one phase/branch per dimension
  6) Nested loops: outer counter first dimension, inner counters next
- If two consecutive counterexamples look similar (same vars at 0/bounds), FORCE switching to the next checklist family.
- Coefficients should be small integers (e.g., -5..5) unless clearly justified by program structure.

NESTED-LOOP PRIORITY RULE (strong priority):
- If there is an inner loop and inside it some variable v updates as v=v-k (k>0 constant) or v--, then prioritize a 2D lex RF:
  dim0 = outer-loop progress (e.g., ReLU(outerVar - 1) or ReLU(outerVar - L))
  dim1 = inner-loop progress (e.g., ReLU(v - 1) or ReLU(v - L))
- Prefer the real inner decreasing variable v (often a local like x1) over counter-based surrogates.

MULTI-LOOP-HEAD RULE:
- If Loop heads count >= 2, prioritize a lex vector aligned per loop head:
  pick one variable that strictly decreases on the backedge of each head,
  then use one dimension per head (2D–6D).

BRANCH/JUMP VECTORIZATION RULE:
- If evidence shows k branches/jumps (e.g., [checkVec] lists k jumps), prefer a k-D (or k+1-D) lex vector.
- One dimension per branch:
  - branch has x-- : use ReLU(x - L)
  - branch has x++ with an upper bound: use ReLU(U - x)
  - swap/reset phases: separate phases with a phase dimension (still affine-ReLU), then add branch dimensions
- Requirement: on any taken branch, at least ONE dimension strictly decreases; all earlier dimensions do not increase.
